## 🎯 Propósito

Este documento define la arquitectura de la interfaz de RemapH V3.

Describe la organización de la UI, la responsabilidad de sus componentes y el flujo actual de interacción.

No describe la lógica del Runtime.

No describe la integración con Windows.

# 🎯 1. Objetivos

La interfaz tiene una única misión:

Permitir al usuario crear, modificar y comprender sus remapeos de la forma más rápida y clara posible.

La UI:

• No ejecuta remapeos.
• No conoce las reglas internas del Runtime.
• No contiene lógica específica de Windows.

La UI trabaja sobre un Perfil editable.

---

# 🏛️ 2. Organización general

La UI se organiza mediante componentes independientes.

```
Aplicación
│
├── Toolbar
├── Tabla
│   ├── Cabecera
│   └── Filas
├── StatusBar
└── Popups
```

En el estado actual no existen ventanas especializadas implementadas.

---

# 🧩 3. Estructura de la interfaz

La pantalla principal se construye desde `ui_layout.ts`.

```
Layout
│
├── Toolbar
├── Tabla
├── StatusBar
└── Contenedor global de Popups
```

---

## 🔹 Toolbar

Responsable de:

• Nombre de la aplicación.
• Selector visual de perfil.
• Estado del perfil.
• Nueva fila.
• Configuración.

En el estado de commit 004 el sistema completo de perfiles todavía está pendiente.

---

## 🔹 Tabla

Responsable de mostrar el Perfil UI.

Está formada por:

• Cabecera.
• Filas.

La definición de columnas vive en `ui_columnas.ts`.

La tabla utiliza la misma definición para construir cabecera y filas.

---

## 🔹 StatusBar

Responsable de mostrar información contextual.

No contiene lógica de remapeo.

---

## 🔹 Popups

Los Popups ofrecen selecciones rápidas.

El contenedor global está en:

`comp_popup_contenedor.ts`

Las opciones concretas se definen en:

`comp_popup_abrir.ts`

---

# 🪪 4. Filas

La fila es la unidad visual principal del programa.

Una fila corresponde exactamente a un `FilaPerfil`.

Cada fila posee:

• ID.
• ContextoFila.
• Celdas.
• Componentes visuales.

El ID se conserva aunque la fila sea reconstruida.

El ContextoFila transporta esa identidad hacia los componentes internos.

---

# 🧱 5. Componentes

En el estado actual los componentes se organizan principalmente por responsabilidad.

## 🟢 Controles de fila

Ejemplos:

• Estado.
• Tipo.
• App.
• Color.
• Ejecución.
• Nota.

Estos controles modifican el `FilaPerfil` correspondiente.

---

## 🎯 Capturador

El capturador muestra el Trigger y permite iniciar una captura.

La captura física y el análisis se separan en:

• `comp_capturador.ts`
• `comp_capturador_trigger.ts`

---

## ⚙️ Acción

`comp_accion.ts` decide qué componente visual representa la acción según el tipo de fila.

---

## ⚪ Componentes neutros

Ejemplos:

• Layout.
• Tabla.
• Cabecera.
• Contenedores.

Su función principal es organizar la UI.

---

# 🔄 6. Componentes dinámicos

La columna Acción puede cambiar de contenido según el tipo.

Ejemplo:

```
Tipo: Teclado
        ↓
Capturador
```

```
Tipo: Multimedia
        ↓
Control Multimedia
```

```
Tipo: Macro
        ↓
Control Macro
```

El componente visual se construye según el tipo actual.

La decisión se concentra en `comp_accion.ts`.

---

# 👆 7. Flujo de interacción

El flujo actual de edición es:

```
Usuario
↓
Control UI
↓
FilaPerfil
↓
Reconstrucción visual
```

Cuando una operación necesita reconstruir una fila:

```
Componente
↓
ui_tabla_control
↓
ui_tabla
↓
Reconstruir fila
```

La UI no accede directamente al DOM de otro componente para modificarlo.

La reconstrucción es la forma actual de actualizar contenido dependiente.

# 📢 9. Barra de estado

La barra inferior está destinada a información contextual de la aplicación.

No debe utilizarse como sustituto de la tabla ni de los controles.

---

# ✅ 10. Buenas prácticas

Antes de crear un nuevo componente:

¿Tiene una única responsabilidad?

¿Pertenece realmente a la UI?

¿Puede reconstruirse sin romper la fila?

¿Es independiente del Runtime?

¿La lógica pertenece a otro módulo?

La fila es la unidad principal de configuración.

Las ventanas deben reservarse para configuraciones que no caben razonablemente dentro de una fila.
